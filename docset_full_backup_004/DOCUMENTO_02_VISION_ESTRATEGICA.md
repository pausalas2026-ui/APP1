# DOCUMENTO 2 — CAPÍTULO 1: VISIÓN ESTRATÉGICA

## 1.1. Qué es I Love To Help
- Cualquier persona puede convertirse en promotor
- Cualquier causa puede activar red de promotores
- Cualquier producto puede ser vendido por cualquier usuario
- Cualquier compra puede incluir donación directa
- Toda acción genera trazabilidad, comisiones y aportes sociales

## 1.2. Diferencias Clave
- NO es ecommerce tradicional (incluye tracking, causas, MLM, tienda personal)
- NO es sistema de donaciones clásico (donaciones desde flujos comerciales)
- NO es MLM clásico (simple, transparente, solo N1 y N2)
- NO es app de sorteos estándar (integra causas y productos)
- NO es marketplace tradicional (aprobaciones, comisiones, promoción externa)

## 1.3. Objetivo Principal
"Convertir a cualquier persona en una fuerza de ventas y recaudación"

## 1.4. Objetivos Secundarios
- Viralidad y crecimiento orgánico
- Sostenibilidad económica
- Beneficio real para promotores
- Beneficio real para causas
- Beneficio para creadores de productos

## 1.5. Relación entre Componentes
- APP Móvil: acceso rápido, promoción inmediata
- Plataforma Web: administración, dashboard profesional
- Marketplace: carga, aprobación, comisiones
- Sorteos: viralidad, engagement
- Causas: propósito, donación directa
- Motor económico: une todo

## 1.6. Principios Rectores
- Simplicidad UX, complejidad backend
- Escalabilidad estructural
- Modularidad estricta
- Backend unificado para lógica crítica
- Transparencia económica total

## 1.7. Interpretación al Programar
- Cada módulo conectado con al menos otros 4
- Cada acción puede activar comisiones, donaciones, tracking
- Sistema reactivo y relacional (no secuencial)
- Cada endpoint respeta roles y planes
- Cada cálculo pasa por motor universal
